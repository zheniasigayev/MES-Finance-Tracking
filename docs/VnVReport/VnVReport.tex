\section{Comparison to Existing Implementation}	

\subsection{Existing Implementation}
\textbf{Spreadsheet Dependency} - Due to the inefficient nature of the current reimbursement process, many clubs submit a one time spreadsheet with all reimbursements for the semester. The spreadsheet was subject to strict formatting, naming conventions, and meticulous documentation of receipts. All of this was required to ensure that the finance team could process the reimbursements without needing to follow up with clubs for missing information. This process was time consuming and error prone, and it was difficult for clubs to track the status of their reimbursement requests.\\
\textbf{Tedious File Handling} - The current process requires clubs to upload itemized receipts as separate PDFs or images. Because the maximum 10 file upload limit per submissions, clubs with more than 10 receipts must submit multiple expense reports, which adds overhead and complexity to the submission process. Additionally, the finance team must manually download and organize all receipt files for processing, which is inefficient and increases the risk of lost or misfiled documents.\\
\textbf{Manual Verification Process} - Submitters had to manually verify that all dates, taxes, totals, and proof of payment were clearly documented and were cross referenced in the spreadsheet.\\
\textbf{Lack of Visibility} - Once a reimbursement request was submitted, there was no way for submitters to track the status of their requests. This lack of visibility led to uncertainty and increased follow-up inquiries to the finance team, further burdening their workload.\\

\subsection{New Implementation}
\textbf{Elimination of Spreadsheet Dependency} - The new reimbursement management system eliminates the need for spreadsheets by providing a structured web interface for submitting reimbursement requests. The new optical character recognition system can automatically fill in key reimbursement information, which clubs can review and revise for accuracy. This streamlines the submission process and reduces the likelihood of formatting errors. Furthermore, this process is much simpler and allows for clubs to submit reimbursements throughout the year in a much simpler fashion, rather than needing to wait until the end of the semester to submit a large spreadsheet.\\
\textbf{Streamlined File Handling} - The new system allows clubs to upload receipts directly through the web interface. Despite only being allowed to submit receipts one at a time, the system provides a more intuitive and user-friendly experience. The finance team can also view receipts directly from the reimbursement review dashboard without needing to download and organize files manually, improving efficiency and reducing the risk of lost documents.\\
\textbf{Centralized Tracking and Visibility} - The system tracks the status of each expense report and provides visibility to submitters through the dashboard. Submitters can see whether their requests are pending, approved, or rejected, which reduces uncertainty and minimizes follow-up inquiries to the finance team.\\
\textbf{Optimized Reviewer Workflow} - Reviewers are no longer required to manually cross reference spreadsheet and image files. Rather, the images can be seen through the click of a button next to the reimbursement submission itself. This way, the approval process is quick and seamless, allowing for more efficient processing of reimbursements. \\

\section{Changes Due to Testing}

Feedback gathered during usability testing and development resulted in several improvements to the reimbursement management system. These changes focused on improving reviewer usability, streamlining workflows, and increasing system robustness.

\textbf{Improved Reimbursement Review Dashboard} – The dashboard view was updated to display more contextual information to help reviewers quickly identify requests. The interface now shows the club name, submitter name, submission date, transaction date, merchant name, and reimbursement amount. Reviewers can also directly view the uploaded receipt from the dashboard.

\textbf{Integrated Financial Management Interface} – The reimbursement review dashboard was previously implemented as a separate page from the club expenses analytics dashboard. These views were merged into a single tabbed interface on the club expenses page, with \textit{Analytics} and \textit{Review Requests} tabs appearing once a club is selected. This reduces navigation overhead and centralizes financial management tasks.

\textbf{Improved Navigation and Context Selection} – A ``View Past Reimbursements'' link was added to the admin club overview modal, allowing administrators to open the expenses page with the appropriate club automatically selected via a query parameter. Additionally, the system now automatically selects the club when a user belongs to only one club.

\textbf{More Robust Receipt Submission Workflow} – The receipt submission process was updated to better handle OCR parsing failures and incorrect extractions. Users are now able to manually correct receipt information when the automated parser fails or extracts incorrect values.

\textbf{Improved Data Model Clarity} – Database fields were renamed to use clearer and more descriptive names (e.g., \texttt{completedAt}, \texttt{decision}, and \texttt{decisionTime}), improving maintainability and readability of the stored reimbursement data.

\textbf{Refactored Backend Review APIs} – The reimbursement review workflow was refactored to use dedicated API endpoints (\texttt{/api/receipts/admin/requests} and \texttt{/api/receipts/admin/decide}). These endpoints replace the previous server actions and use the existing \texttt{authorizeFinanceUser()} middleware, with requests filtered by the selected club and access controlled using the per-club \texttt{canManageFinances} permission.

\section{Automated Testing}

To improve reliability and maintain code quality, several automated testing and verification measures were integrated into the development workflow.

\begin{itemize}

\item \textbf{Type Checking and Linting} – Automated type checking and linting tools are executed as part of the development workflow to detect type inconsistencies, syntax errors, and violations of coding standards. Lint checking is performed using \texttt{ESLint}, which helps maintain consistent code style and catch potential issues before code is merged.

\item \textbf{Commit Message Format Verification} – Automated checks validate that commit messages follow the required formatting conventions. Enforcing consistent commit messages improves traceability of changes and ensures that project history remains clear and well documented.

\item \textbf{Automated Unit Testing} – Unit tests are implemented using the \texttt{Jest} testing framework. These tests verify the correctness of individual components and functions, and are executed automatically to ensure that new changes do not introduce regressions.

\end{itemize}
